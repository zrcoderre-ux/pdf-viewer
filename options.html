How the viewer should name a PDF in the toolbar and on download.

The footer option reads the document title printed at the bottom of court filings and applies legal-document abbreviation rules (Declaration → Decl., Opposition → Opp., etc.). When two open PDFs have the same derived name, the viewer disambiguates them automatically (e.g. Demurrer becomes Demurrer to SAC and Demurrer to FAC). The source-filename option uses the original name as the server sent it — no transformation.

Extra PDF URL patterns

Some sites serve PDFs without a .pdf in the URL (LA Court eCMS, agency document portals, etc.). Add URL patterns here and the extension will treat matching URLs as PDFs. Use * as a wildcard. One pattern per line.

Examples:

https://civil.lacourt.org/ecourt/ecms/doc*
https://www.courts.ca.gov/opinions/*
https://example.gov/getfile?id=*

Note: The LA Superior Court eCMS pattern (civil.lacourt.org/ecourt/ecms/doc*) is built in — you don't need to add it here.

OCR settings

When a scanned PDF has no selectable text, OCR (Tesseract) recognizes it so the text becomes selectable and citations are linked. By default this runs only when you click the OCR button in the viewer toolbar; enable auto-OCR below to run it automatically on every scanned document.

Use the setting below to ignore text found in the far-left margin — useful for law firm names and phone numbers printed in the gutter of court filings.

0 = no exclusion. 8 (default) skips roughly the leftmost half-inch of a letter-size page. Maximum 30.
